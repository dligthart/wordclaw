Designed for governed automation. Core Capabilities

WordClaw helps agents work on governed content, assets, public intake flows, and paid-access contracts without losing human oversight or runtime safety.

🧭

Deployment Discovery

Start with capability, status, identity, and workspace discovery so agents can see the active transport, auth posture, domain state, onboarding path, and best next target before writing.

🧱

Governed Content Runtime

Collections, singleton globals, localization, preview tokens, reverse references, and publication-state aware reads sit behind one policy and audit model.

📊

Content Runtime Queries

Use schema-aware field filters, grouped projections, and lifecycle-aware reads for leaderboard, analytics, and session-style content models.

🖼️

Schema-Aware Assets

Store assets as first-class records with local or S3-compatible backends, signed or entitled delivery, and direct-provider upload flows.

🧾

Forms + Public Write Lanes

Support bounded external writes through reusable forms, short-lived write tokens, optional workflow transitions, and follow-up webhook or job execution.

⚙️

Background Jobs

Queue webhook delivery, deferred transitions, and generic domain-scoped work with worker health visibility instead of ad hoc side processes.

🔌

REST + Reactive MCP

Expose the runtime through deterministic REST contracts and remote MCP sessions that can subscribe to review, content, and integration events in real time.

🛡️

Workflow + Approvals

Route content changes through review gates so agents can draft and propose safely while humans retain final control over sensitive transitions.

💳

L402 + Entitlements

Run Lightning-gated offers, entitlement purchases, and paid reads or asset access as part of the core runtime contract.

Experimental Modules Stay Scoped

Delegation and broader platform experiments remain intentionally outside the default supported WordClaw path.

  • Agent runs: gated behind `ENABLE_EXPERIMENTAL_AGENT_RUNS`.
  • Monetization experiments: AP2 and payout expansion stay in RFC history or optional code paths.
  • Rule of thumb: build against REST, MCP, and the Tier 1 feature set first.

Built for Governed Automation

WordClaw is now more than schema CRUD. The live runtime combines deployment discovery, lifecycle-aware content reads, first-class asset delivery, reusable intake flows, and reactive agent coordination in one governable control plane.

  • Backend: Node.js, Fastify, Drizzle ORM
  • Database: PostgreSQL with pgvector
  • Discovery Layer: capabilities, deployment status, identity, workspace context, and target resolution
  • Content Runtime: schema refs, field-aware queries, grouped projections, public write lanes, forms, jobs, and TTL lifecycle archival
  • Asset Layer: local or S3-compatible storage with multipart and direct upload, signed delivery, entitlement checks, and variants
  • Control Plane: SvelteKit supervisor UI for audits, schemas, approvals, jobs, forms, assets, and payments
  • Agent Surface: REST, MCP, CLI guidance, and remote reactive subscriptions

What ships today?

Core runtime, assets, forms, background jobs, paid access, and supervisor tooling are shipped on `main`. Reactive MCP and vector-RAG ergonomics are the main hardening areas.
